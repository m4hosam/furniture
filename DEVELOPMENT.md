## 1. Tech Stack

| Concern | Tool |
|---|---|
| Framework | React 19 (Vite) |
| Styling | Tailwind CSS v4 (`@import "tailwindcss"`) |
| Canvas rendering | SVG (via JSX) |
| Persistence | Browser `localStorage` |
| Fonts | Inter (Google Fonts) |

No external UI libraries, no state management library, no backend.

## 3. Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│  App.jsx  (orchestrator)                                     │
│                                                             │
│  useApartment ──┐                                           │
│  useElements ───┼──► <Sidebar>    (pure display + inputs)   │
│  useFileIO  ────┘                                           │
│                                                             │
│  useElements ───┬──► <Canvas>     (SVG rendering)           │
│                 └──► useDrag      (pointer engine, internal) │
└─────────────────────────────────────────────────────────────┘
```

**Key principle:** All state lives in hooks. All components are "dumb" — they receive data via props and fire callbacks. No component holds its own business-logic state.

## 4. Data Model

### `apartment` object

```js
{
  width: number,   // Total floor width in cm (SVG user units)
  depth: number,   // Total floor depth in cm
}
```

Persisted to `localStorage` under key `"apartmentConfig"`.

---

### `Element` object

The `elements` array is the single source of truth for everything on the canvas — rooms, furniture, doors, and windows are all the same shape.

#### Rectangular element (`shape: 'rect'`)

```js
{
  id:       string,            // e.g. "a1b2c3d4e" — generated by generateId()
  type:     'room' | 'furniture' | 'door' | 'window',
  shape:    'rect',
  name:     string,            // Display label shown on canvas and in layers panel
  x:        number,            // Position of top-left corner in SVG user units (cm)
  y:        number,
  w:        number,            // Width in cm
  h:        number,            // Height (depth) in cm
  parentId: string | null,     // ID of the room this element belongs to, or null
}
```

#### Polygon element (`shape: 'polygon'`)

```js
{
  id:       string,
  type:     'room',            // Currently only rooms can be polygons
  shape:    'polygon',
  name:     string,
  x:        number,            // Offset of the entire polygon group
  y:        number,
  points:   Array<{ x: number, y: number }>, // Relative to (x, y)
  parentId: null,              // Polygonal rooms are always top-level
}
```

> **Z-Index:** Controlled by array order. Elements drawn last are visually on top.  
> `bringToFront` moves an element to the **end** of the array.  
> `sendToBack` moves it to the **start**.

## 5. Custom Hooks

### `useApartment`

**File:** `src/hooks/useApartment.js`

Manages the global floor dimensions and auto-saves them.

```js
const { apartment, setApartment } = useApartment();
```

| Return | Type | Description |
|---|---|---|
| `apartment` | `{ width, depth }` | Current floor dimensions |
| `setApartment` | `Function` | Standard React setter — pass a full object |

**localStorage key:** `"apartmentConfig"`

---

### `useElements`

**File:** `src/hooks/useElements.js`

The core hook. Manages the entire elements array, selection, CRUD, and layer ordering.

```js
const {
  elements, setElements,
  selectedId, setSelectedId, selectedEl,
  rooms,
  addRectElement, addCustomRoom,
  updateSelected, rotateSelected,
  deleteElement,
  moveElementLayer, bringToFront, sendToBack,
} = useElements();
```

#### Returned values

| Name | Type | Description |
|---|---|---|
| `elements` | `Element[]` | The full elements array (ordered = z-order) |
| `setElements` | `Function` | Raw setter — use only for bulk operations (e.g. import) |
| `selectedId` | `string \| null` | ID of the currently selected element |
| `setSelectedId` | `Function` | Set selection programmatically |
| `selectedEl` | `Element \| null` | The selected element object (derived, never stale) |
| `rooms` | `Element[]` | Filtered subset — only `type === 'room'` |
| `addRectElement(type, w, h)` | `Function` | Adds a new rect element, auto-selects it |
| `addCustomRoom(w, h)` | `Function` | Adds the default L-shape polygon room |
| `updateSelected(key, value)` | `Function` | Updates one field on the selected element |
| `rotateSelected()` | `Function` | Swaps `w` and `h` on the selected rect element |
| `deleteElement(id)` | `Function` | Removes element, un-parents its children |
| `moveElementLayer(index, 'up' \| 'down')` | `Function` | Swaps adjacent elements in the array |
| `bringToFront(id)` | `Function` | Moves element to end of array |
| `sendToBack(id)` | `Function` | Moves element to start of array |

**localStorage key:** `"apartmentElements"`

#### How `deleteElement` handles orphaned children

When a room is deleted, any element with `parentId === deletedId` has its `parentId` set to `null` before removal. Children are not deleted — they become free-floating on the apartment canvas.

---

### `useDrag`

**File:** `src/hooks/useDrag.js`

Encapsulates the SVG Pointer Events drag engine. Used internally by `Canvas/index.jsx`.

```js
const { handlePointerDown, handlePointerMove, handlePointerUp } = useDrag(
  elements,     // Element[] — read-only, for coordinate lookups
  setElements,  // Functional setter — mutates positions during drag
  svgRef,       // React ref to the <svg> element
);
```

#### How coordinate conversion works

SVG elements are rendered in "user space" (cm), but pointer events deliver pixel coordinates relative to the browser viewport. The hook uses the browser's `SVGPoint.matrixTransform()` API to apply the inverse of the SVG's current transform matrix:

```js
const pt = svgRef.current.createSVGPoint();
pt.x = e.clientX;
pt.y = e.clientY;
const svgCoords = pt.matrixTransform(svgRef.current.getScreenCTM().inverse());
```

This makes dragging work correctly regardless of viewport scaling, zoom, or any CSS transforms on the SVG.

#### Drag types

| `dragType` | Triggered by | Behaviour |
|---|---|---|
| `'element'` | `onPointerDown` on any shape | Moves the element and all its children together |
| `'vertex'` | `onPointerDown` on a vertex circle | Moves a single polygon point |

**Pointer capture** (`e.target.setPointerCapture`) is used so the drag continues even if the mouse leaves the element's hit area during fast movement.

---

### `useFileIO`

**File:** `src/hooks/useFileIO.js`

Handles JSON save/load and layout wipe.

```js
const { fileInputRef, exportToJson, importFromJson, triggerFileInput, resetLayout } =
  useFileIO({ apartment, elements, setApartment, setElements, setSelectedId });
```

#### Export format

```json
{
  "apartment": { "width": 1000, "depth": 800 },
  "elements": [ ...Element[] ]
}
```

The `fileInputRef` must be attached to a `<input type="file" ref={fileInputRef} accept=".json" />` element somewhere in the JSX (done inside `FileSync.jsx`).

## 6. Utility Functions

**File:** `src/utils/geometry.js`

These are pure functions with no React dependency — easy to unit test.

### `generateId() → string`

Returns a 9-character random alphanumeric string. Used as element IDs.

```js
import { generateId } from '../utils/geometry';
const id = generateId(); // → "a3f9x2k7m"
```

### `getPolygonCenter(points) → { cx, cy }`

Calculates the center of a polygon's bounding box. Used to position the label text inside polygon rooms.

```js
getPolygonCenter([{ x: 0, y: 0 }, { x: 300, y: 0 }, { x: 300, y: 150 }, { x: 0, y: 150 }])
// → { cx: 150, cy: 75 }
```

### `getElementStyle(type) → { fill, stroke, strokeWidth }`

The single source of truth for visual styling by element type. Returns raw SVG style values.

```js
getElementStyle('furniture')
// → { fill: 'rgba(191, 219, 254, 0.9)', stroke: '#3b82f6', strokeWidth: 2 }
```

> **When adding a new element type**, this is the first function to update.

## 7. Components

### `App.jsx` — Orchestrator

The top-level component. Its only job is to call hooks and pass the resulting values into `<Sidebar>` and `<Canvas>` as props. It contains **no JSX logic, no conditionals, no event handlers**.

```
App
├── useApartment()      → apartment, setApartment
├── useElements()       → elements, selectedId, CRUD functions...
├── useFileIO(...)      → export/import/reset
├── <Sidebar .../>      → receives all sidebar-relevant props
└── <Canvas  .../>      → receives canvas-relevant props
```

---

### Sidebar Components

All sidebar components are **stateless functional components** that receive props and fire callbacks.

#### `Sidebar/index.jsx`

The sidebar container with a dark branded header. Composes all sub-panels.

**Props:**

| Prop | Type | Description |
|---|---|---|
| `fileInputRef` | `ref` | Forwarded to `<FileSync>` |
| `onExport` | `() => void` | Triggers JSON download |
| `onImport` | `(e) => void` | Handles file input change event |
| `onTriggerImport` | `() => void` | Programmatically clicks the file input |
| `onReset` | `() => void` | Shows confirm dialog then resets |
| `apartment` | `object` | `{ width, depth }` |
| `onApartmentChange` | `(apartment) => void` | Called when width/depth changes |
| `elements` | `Element[]` | Full element list |
| `selectedId` | `string \| null` | Currently selected ID |
| `selectedEl` | `Element \| null` | Currently selected element |
| `rooms` | `Element[]` | Filtered rooms array |
| `onSelect` | `(id) => void` | Sets selection |
| `onAddRect` | `(type) => void` | Adds a rect element |
| `onAddCustomRoom` | `() => void` | Adds an L-shape polygon room |
| `onUpdate` | `(key, value) => void` | Updates selected element field |
| `onRotate` | `() => void` | Rotates selected element 90° |
| `onDelete` | `(id) => void` | Deletes an element |
| `onBringToFront` | `(id) => void` | Moves to top of z-order |
| `onSendToBack` | `(id) => void` | Moves to bottom of z-order |
| `onMoveLayer` | `(index, dir) => void` | Swaps adjacent layers |

#### `FileSync.jsx`

Export and Import JSON buttons + hidden `<input type="file">`.

#### `ApartmentSettings.jsx`

Width and Depth number inputs. Reset All button. Uses `htmlFor`/`id` pairs for accessibility.

#### `BuildTools.jsx`

Add-element button toolbar. Buttons are type-coloured:
- **Rooms** → slate
- **Furniture** → blue (primary CTA)
- **Door** → amber
- **Window** → sky

#### `PropertiesPanel.jsx`

Conditionally renders when `selectedEl` is not null. Shows:
- Name input
- "Attach to Room" select (hidden for rooms)
- Width/Depth inputs (hidden for polygons, shows hint instead)
- Rotate 90° (hidden for polygons)
- Delete, Bring to Front, Send to Back

#### `LayersPanel.jsx`

Lists all elements in reverse draw order (top of list = visually on top). Each row shows:
- Colour dot + name
- "in Room Name" sub-label (if has parent)
- Type badge chip (Room / Furniture / Door / Window)
- Layer up ↑ / Layer down ↓ / Delete × buttons

---

### Canvas Components

#### `Canvas/index.jsx`

Instantiates `useDrag`, renders the `<svg>`, handles background deselect.

**Props:** `apartment`, `elements`, `setElements`, `selectedId`, `setSelectedId`

#### `CanvasGrid.jsx`

Renders two SVG `<pattern>` definitions:
- `grid-minor` — 50cm grid (light `#e2e8f0` lines)
- `grid-major` — 200cm grid (slightly heavier `#cbd5e1` lines)

No props required except `width` and `height`.

#### `CanvasElement.jsx`

Renders a single element. Handles rect vs polygon branching, selection glow, label text, and vertex drag handles.

**Props:**

| Prop | Type | Description |
|---|---|---|
| `el` | `Element` | The element to render |
| `isSelected` | `boolean` | Whether this element is selected |
| `onPointerDown` | `(e, 'element', id) => void` | Called when shape body is clicked |
| `onVertexPointerDown` | `(e, 'vertex', id, vIndex) => void` | Called when a polygon vertex circle is clicked |

**Selection visual:** A soft semi-transparent glow rect/polygon is drawn behind the element when `isSelected` is true. For non-room rects, a dashed stroke is also applied.

## 8. How Data Flows

```
User clicks "+ Add Furniture"
  → BuildTools calls props.onAddRect('furniture')
    → App passes: onAddRect={(type) => addRectElement(type, apartment.width, apartment.depth)}
      → useElements.addRectElement creates new element, appends to array, sets selectedId
        → elements state updates → React re-renders Canvas + Sidebar

User drags a Sofa on the canvas
  → CanvasElement fires onPointerDown → Canvas/index calls useDrag.handlePointerDown
    → dragInfo is set with start coords + children snapshot
      → pointermove fires → handlePointerMove updates elements positions
        → elements state updates → Canvas re-renders with new positions

User changes "Label / Name" input
  → PropertiesPanel fires onUpdate('name', newValue)
    → App passes: onUpdate={updateSelected}
      → useElements.updateSelected patches selected element
        → elements state updates → LayersPanel + CanvasElement re-render with new name
```

## 9. localStorage Persistence

Two keys are used:

| Key | Hook | Content |
|---|---|---|
| `"apartmentConfig"` | `useApartment` | `{ width, depth }` |
| `"apartmentElements"` | `useElements` | `Element[]` |

Both are written on every state change via `useEffect`. On page load, both hooks initialize from storage if present, otherwise fall back to hardcoded defaults.

**Reset** (`resetLayout`) wipes both keys and calls `window.location.reload()` to re-initialize from defaults.

## 10. Extension Recipes

### Add a New Element Type

Say you want to add a **"staircase"** type.

**Step 1:** Register default sizes in `useElements.js`:
```js
// Inside addRectElement, add to the defaults object:
staircase: { w: 100, h: 200, name: 'Staircase', x: cx - 50, y: cy - 100 },
```

**Step 2:** Add visual styles in `utils/geometry.js`:
```js
case 'staircase':
  return { fill: '#fef3c7', stroke: '#d97706', strokeWidth: 3 };
```

**Step 3:** Add a button in `Sidebar/BuildTools.jsx`:
```jsx
<button onClick={() => onAddRect('staircase')} className="...">
  + Staircase
</button>
```

**Step 4:** Add a type badge in `Sidebar/LayersPanel.jsx`:
```js
const TYPE_LABEL = {
  // ...existing types...
  staircase: { text: 'Stair', cls: 'bg-yellow-50 text-yellow-700 border-yellow-100' },
};
```

That's it — the canvas renderer, properties panel, layers panel, drag engine, and JSON export/import all work automatically because they treat all elements generically.

---

### Add a New Sidebar Panel

1. Create `src/components/Sidebar/MyPanel.jsx` as a stateless component.
2. Add any new state/logic needed to the appropriate hook (or create a new one).
3. Import and render `<MyPanel>` inside `Sidebar/index.jsx`.
4. Pass the required props down from `App.jsx`.

---

### Add a New Canvas Shape

Currently elements are either `shape: 'rect'` or `shape: 'polygon'`. To add a third shape (e.g. `'circle'`):

1. Add a new branch in `CanvasElement.jsx`:
```jsx
{el.shape === 'circle' && (
  <circle
    cx={el.r} cy={el.r} r={el.r}
    fill={fill} stroke={activeStroke} strokeWidth={activeStrokeWidth}
    {...interactionProps}
  />
)}
```
2. Make sure `useDrag.handlePointerMove` handles the new shape's position update if its geometry differs from rect (`x`, `y`, `w`, `h`).
3. Add a creation helper in `useElements.js`.

---

### Add Snapping / Grid Locking

The drag delta is calculated in `useDrag.handlePointerMove`. To snap to a 50cm grid, modify the position update:

```js
const snap = (val, grid = 50) => Math.round(val / grid) * grid;

// In handlePointerMove:
return { ...el, x: snap(dragInfo.startX + dx), y: snap(dragInfo.startY + dy) };
```

You could expose a `snapEnabled` boolean from a new `useSettings` hook and toggle snapping from a toolbar button.

## 11. Design System

The app uses **Tailwind CSS v4** utility classes throughout. Key tokens in use:

| Token | Usage |
|---|---|
| `slate-*` palette | Neutral UI chrome, room styling |
| `blue-600` | Primary CTA (Add Furniture), selection highlight |
| `amber-*` | Door elements |
| `sky-*` | Window elements |
| `red-*` | Danger actions (Delete, Reset All) |
| `Inter` | All text via Google Fonts |

**`src/index.css`** adds:
- `@import "tailwindcss"` — Tailwind v4 entry
- Google Fonts `Inter` import
- `.sidebar-scroll` — custom thin scrollbar for the sidebar
- Base `html/body/#root` height: `100%` to enable full-height layouts

All button variants are defined inline as Tailwind class strings in each component. If buttons become inconsistent, consider extracting a `Button.jsx` component with a `variant` prop.

## 12. Running Locally

```bash
# Install dependencies
npm install

# Start development server (hot reload)
npm run dev

# Lint
npm run lint

# Build for production
npm run build
```

The dev server starts on `http://localhost:5173` (or the next available port).

No environment variables are required. The app is fully client-side.
